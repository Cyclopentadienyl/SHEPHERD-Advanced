# F.5 Workspace Restructure — Engineering Blueprint
# ==================================================
# Temporary working document. Delete after PR merge.
#
# Goal: Unify data/ and models/ into workspace-based layout
# where one workspace = one KG version + all its checkpoints.
#
# Constraint: Core inference/training logic (pipeline.py, trainer.py,
# loss_functions.py) must NOT change. Only path defaults and directory
# creation logic change.
#
# All UI text in English.

# =====================================================
# STEP 1: Directory Structure Definition
# =====================================================
#
# BEFORE (current):
#   data/
#   ├── external/           ← keep (download area)
#   ├── processed/          ← remove (replaced by workspaces)
#   │   └── knowledge_graph/  ← remove (metadata.json, VERSION are empty)
#   └── demo/               ← move to workspaces/demo/
#
#   models/
#   ├── checkpoints/        ← remove (absorbed into workspace)
#   ├── experiments/        ← remove (absorbed into workspace)
#   ├── pretrained/         ← keep (KG-independent, e.g. ClinicalBERT)
#   └── production/         ← remove (workspace manages its own best model)
#
# AFTER:
#   data/
#   ├── external/           ← unchanged
#   └── workspaces/
#       ├── demo/           ← dev/test workspace
#       │   ├── kg.json
#       │   ├── node_features.pt
#       │   ├── edge_indices.pt
#       │   ├── num_nodes.json
#       │   ├── shortest_paths.pt
#       │   ├── shortest_paths.meta.json
#       │   └── checkpoints/
#       │       └── model_checkpoint.pt
#       └── (future: primekg_2026Q1/, etc.)
#
#   models/
#   └── pretrained/         ← only this survives
#
# =====================================================
# STEP 2: Files to Modify (path defaults)
# =====================================================
#
# File                                    | What changes
# ----------------------------------------|------------------------------------------
# configs/deployment.yaml                 | paths.workspaces_root: data/workspaces/

# scripts/train_model.py                  | data_dir default → data/workspaces/default
#                                         | checkpoint_dir default → {data_dir}/checkpoints
# scripts/compute_shortest_paths.py       | help text example paths only
# src/api/routes/pipeline.py              | DEFAULT_DATA_DIR → data/workspaces/default
# src/webui/components/diagnosis_panel.py | saved config default → data/workspaces/default
# src/webui/components/training_console.py| data path default display
# src/api/services/training_manager.py    | checkpoint_dir derivation
# .gitignore                              | update patterns for new layout
